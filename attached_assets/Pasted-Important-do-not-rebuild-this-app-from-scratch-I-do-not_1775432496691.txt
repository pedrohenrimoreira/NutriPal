9. LIQUID GLASS IMPLEMENTATION MANUAL, FOLLOW THIS
==================================================

Implement the UI following iOS Liquid Glass visual behavior.

Mandatory rules:
1. Apply Liquid Glass only to floating interface elements such as header, tab bar, bottom actions, main buttons, pills, toggles, short modals, highlight cards and overlays.
2. Do not use Liquid Glass as the background of the whole screen, in full lists, or in long text blocks.
3. Guarantee legibility. Text and icons on glass must keep proper contrast on both light and dark backgrounds.
4. Prefer native iOS effect when available. On unsupported platforms or versions, use a similar fallback with light blur, translucency, subtle border and minimal shadow.
5. Avoid too many blurred components on the same screen. Prioritize only a few high-impact elements.
6. In animations, do not hide the effect through invalid opacity behavior if the native material does not support it. Use proper material animation or safe fallback.
7. Always respect safe areas, native bars, iOS spacing and navigation hierarchy.
8. Glass must feel functional, not decorative. Every glass element must have a clear navigation, action or focus role.
9. Test behavior in scroll, in light and dark backgrounds, and in headers, checking title and button contrast.
10. If there is doubt about where to apply the effect, apply it only to navigation controls and priority actions.

My recommended implementation references:
- official Apple Liquid Glass behavior and HIG Materials
- iOS native materials whenever possible
- React Native implementation through Callstack Liquid Glass or Expo GlassEffect
- fallback through Expo BlurView
- advanced cases through React Native Skia
- verify real bugs and limitations with Apple Forums and GitHub issues
